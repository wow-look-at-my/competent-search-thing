# watcher-scale design plan (locked 2026-07-17)

CONTRACT (user invariant): identical final index state and feature surface on every tier; tiers differ ONLY in update latency. The sweeper is the unconditional baseline covering every root at all times; fanotify/inotify overlay it as latency reducers. Nothing silently uncovered; startup logs the active tier + convergence bound.

Event model: every signal from any tier is "path dirty" -> reconcile by lstat:
- dirty file: lstat -> exists ? Manager.Add : Manager.Remove
- dirty dir: shallow readdir diff vs the index's children of that dir; recurse ONLY into children new to the index (full add-walk); Remove children gone from disk. Order-independent by construction (required by fanotify mask merging; also removes the ordered-batch dependency the fsnotify path had).

Tiers:
1. fanotify (linux, CAP_SYS_ADMIN, runtime-probed): one group FAN_CLASS_NOTIF|FAN_REPORT_DFID_NAME|FAN_CLOEXEC|FAN_NONBLOCK; one FAN_MARK_FILESYSTEM mark per real (non-virtual/network, non-skipped) superblock under the roots; mask FAN_CREATE|FAN_DELETE|FAN_MOVED_FROM|FAN_MOVED_TO|FAN_ONDIR. 5.9+ baseline; FAN_RENAME deliberately unused (no index move primitive; delete+create converges identically). Per-sb probe: EPERM/ENODEV(null fsid)/EXDEV -> that sb falls to tier 2. fsid -> mount-fd routing for open_by_handle_at; handle->path LRU cache invalidated on dir delete/move; ESTALE = parent already gone = drop event. FAN_Q_OVERFLOW -> sweep request. New/removed superblocks detected via the sweeper's mount-table re-read -> mark/unmark live.
2. inotify hot-set via fsnotify (all platforms; the only tier on darwin/windows, where behavior remains watch-everything): budget = watcher.maxWatches (0=auto: min(max_user_watches/2, 65536) on linux; unlimited elsewhere). Fill priority: roots, then $HOME subtree breadth-first, then shallowest-first; promotion on any dirty-dir activity and on sweep-detected changes; LRU eviction at budget. At budget the backend STOPS issuing adds (no failing-syscall storms); evicted/never-watched dirs stay sweep-covered.
3. Sweeper (always on; watcher.sweepDisabled=true logs a LOUD warning that convergence is now full-rescan-only): default interval 20m (watcher.sweepMinutes, 0=default). Chunked enumeration of indexed dirs (new Manager paging API releases the lock between chunks); per dir: lstat; mtime >= watermark (last completed sweep start) or unknown dir -> shallow reconcile. Also re-reads /proc/self/mounts each pass; mounts appearing/disappearing under roots force-dirty that subtree (closes the mount-onto-existing-dir mtime gap). Throttled stats/sec; ctx-abortable; MinGap coalescing like the Rescanner. Known documented limit: mtime-backdated writes (tar --preserve into an existing dir) converge at the deep rescan (existing Rescanner, unchanged) or manual !rescan.

Overflow on any tier -> sweep request (coalesced). Stats gains Backend + per-tier counts + sweep age; one startup summary line announces tier + bound. OnDegraded reserved for real degradation.

New Manager/Store API (additive only): paged live-dir enumeration + ChildrenOf(dir) via the existing children map + Name/IsDir accessors.

Config: new "watcher" section {backend:"auto"(auto|fanotify|inotify|none), maxWatches:0, sweepMinutes:0, sweepDisabled:false, watchExcludes:[]}; Normalize repairs invalids. watchExcludes = never live-watched, still indexed + swept (staleness = sweep bound). Default index excludes gain: .hg .svn __pycache__ .mypy_cache .pytest_cache .ruff_cache .tox .nox .venv (bare-name patterns, same class as existing .git/node_modules/.cache). Migration rootsVersion 2->3: excludes still containing all of {.git,node_modules,.cache} get the missing noise patterns appended + MigrationNotes; curated/emptied lists get stamp-only + an informational note. schemas/config.schema.json + lockstep schema tests in the same commit. README: tier table, consistency model, setcap cap_sys_admin,cap_dac_read_search grant story + security note. CLAUDE.md architecture map updated.

Tests: pure fanotify buffer-parser tests (synthetic DFID_NAME records, merged masks, overflow); capability-gated fanotify integration (probe-mark a scratch fs; /dev/shm on the dev box; skip in CI); cross-tier equivalence replay (same mutation storm -> identical final index under fanotify [gated] / hot-set with forced tiny budget + eviction / sweep-only); hot-set unit tests (budget/LRU/promotion); sweeper tests (mtime short-circuit, backdating documented-miss, deletion diff via children, mount force-dirty via seam); existing ordered-batch test pins consciously renegotiated to convergence pins (each listed in the PR body).

Measurements: env-gated harness (committed before the redesign) captures BEFORE on the same yardstick as AFTER: registration wall/CPU, watch count (fdinfo ground truth), est kernel bytes (1284 B/watch), storm CPU + convergence, idle CPU, teardown; fanotify tier measured on /dev/shm; real-machine (18.6M entries) extrapolation table in the PR body.
