# Vendored file-icons pack (fonts + generated mapping)

Per-file-type icons for file result rows, from the user-chosen
file-icons/atom pack: font glyphs colored by CSS, with the pack's own
dark/light theme colors.

- Upstream pack: https://github.com/file-icons/atom, pinned commit
  `28520868aee66e576145a0b18aa2cde5444a897a` (config.cson,
  styles/{icons,colours,mixins}.less, fonts/*.woff2).
- Octicons font: https://github.com/primer/octicons at tag `v4.4.0`
  (commit `62c672732695b6429678bcd321520c41af109475`),
  build/font/octicons.woff2 -- the "Octicons Regular" face
  icons.less references but Atom used to supply.
- Licenses and the devopicons EXCLUSION: see LICENSES.md.

## Contents

- `fonts/` -- the four licensed woff2 faces (binary assets, ASCII
  rule exempt like PNGs).
- the mapping artifact `internal/fileicons/data.bin` (repo root; it
  lives beside its Go decoder) -- generated by `tools/convert.mjs`
  (see the header comment for the exact semantics ported from the
  pack: suffix/regex match rules in priority-desc order, auto-X
  colour expansion, the lightness(background)<50% mixin evaluation
  producing a [darkTheme, lightTheme] hex pair per rule, matchPath
  and unlicensed-face rules dropped, every glyph cmap-verified). A
  binpazer container (wow-look-at-my/bin-file-fmt) holding one
  IconRules block; the payload encoding is documented in
  `tools/emitbin.mjs` and decoded Go-side by internal/fileicons.
- `tools/convert.mjs` -- the converter; `tools/emitbin.mjs` -- the
  data.bin emitter (payload encoder + the first-party `binpazer`
  CLI pack/validate pipeline); `tools/woff2cmap.mjs` -- the WOFF2
  cmap reader convert.mjs uses, format 4 glyph-aware + format 12,
  cross-checked against fontTools on all four fonts (equal mod
  sub-0x100 mac-subtable controls; internal/fileicons' Go test
  helper ports it for the committed-data gate).

## Regenerating

Needs the `binpazer` CLI (download from pazer.build project
`bin-file-fmt/binpazer`, or build wow-look-at-my/bin-file-fmt with
`cd go && go-toolchain`; `$BINPAZER` overrides the PATH lookup):

    git clone https://github.com/file-icons/atom && git -C atom checkout <pin>
    git clone https://github.com/primer/octicons && git -C octicons checkout v4.4.0
    node frontend/src/fileicons/tools/convert.mjs atom octicons \
      frontend/src/fileicons internal/fileicons/data.bin

Then re-run the integrity tests (go-toolchain + npm test) and update
the pins above.
