# Vendored file-icons pack (fonts + generated mapping)

Per-file-type icons for file result rows, from the user-chosen
file-icons/atom pack: font glyphs colored by CSS, with the pack's own
dark/light theme colors.

- Upstream pack: https://github.com/file-icons/atom, pinned commit
  `28520868aee66e576145a0b18aa2cde5444a897a` (config.cson,
  styles/{icons,colours,mixins}.less, fonts/*.woff2).
- Octicons font: https://github.com/primer/octicons at tag `v4.4.0`
  (commit `62c672732695b6429678bcd321520c41af109475`),
  build/font/octicons.woff2 -- the "Octicons Regular" face
  icons.less references but Atom used to supply.
- Licenses and the devopicons EXCLUSION: see LICENSES.md.

## Contents

- `fonts/` -- the four licensed woff2 faces (binary assets, ASCII
  rule exempt like PNGs).
- `data.json` -- generated by `tools/convert.mjs` (run it against
  the two pinned checkouts; see the header comment for the exact
  semantics ported from the pack: suffix/regex match rules in
  priority-desc order, auto-X colour expansion, the
  lightness(background)<50% mixin evaluation producing a
  [darkTheme, lightTheme] hex pair per rule, matchPath and
  unlicensed-face rules dropped, every glyph cmap-verified). An
  internal single-party format -- deliberately NO schema in
  schemas/, the history.json stance. Pure-ASCII bytes (non-ASCII
  regex sources are \u-escaped by JSON).
- `tools/convert.mjs` -- the converter; `tools/woff2cmap.mjs` -- the
  WOFF2 cmap reader it (and the vitest integrity test) use, format
  4 glyph-aware + format 12, cross-checked against fontTools on all
  four fonts (equal mod sub-0x100 mac-subtable controls).

## Regenerating

    git clone https://github.com/file-icons/atom && git -C atom checkout <pin>
    git clone https://github.com/primer/octicons && git -C octicons checkout v4.4.0
    node frontend/src/fileicons/tools/convert.mjs atom octicons frontend/src/fileicons

Then re-run the integrity tests and update the pins above.
